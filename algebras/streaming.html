Streaming

This algebra provides vocabulary to describe WebSockets and endpoints whose responses are streamed.

API documentation

Chunked Endpoints

The Streaming module enriches the Endpoints algebra with an operation for defining endpoints whose responses produce a stream of values.

For instance, you can define an endpoint streaming a binary file as follows:

val logo: ChunkedEndpoint[Unit, Array[Byte]] =
  chunkedEndpoint(get(path / "logo.png"), bytesChunks())
StreamingDocs.scala

The return type, ChunkedEndpoint[Unit, Array[Byte]], type means an endpoint whose request takes no parameter (Unit) and whose response produces a stream of Array[Byte] values.

Conceptually, the type ChunkedEndpoint[A, B] is equivalent to Endpoint[A, Chunks[B]], where the type Chunks[A] means a stream of values of type A. Please refer to the documentation of your interpreter to know the concrete semantic type given to ChunkedEndpoint.

Responses are streamed using the chunked transfer-encoding, which is supported by most HTTP 1.x clients and servers.

WebSockets

Bidirectional communication is possible via WebSockets. For instance, here is how you can define a WebSocket endpoint for joining a chat room:

val chat: WebSocketEndpoint[Unit, String, String] =
  webSocketEndpoint(path / "chat", textChunks(), textChunks())
StreamingDocs.scala

The return type, WebSocketEndpoint[Unit, String, String], means a WebSocket endpoint whose request takes no parameter, and which yields a channel receiving String messages and sending String messages.

JsonStreaming

Clients and servers have to agree on the serialization format used by response chunks and WebSocket messages. The JsonStreaming module provides a jsonChunks constructor for defining streams of values that are serialized into the JSON text format:

trait JsonStreamingExample
  extends endpoints.algebra.JsonStreaming
    with endpoints.algebra.JsonSchemaEntities {

  val ticks = chunkedEndpoint(get(path / "ticks"), jsonChunks[Unit]())

}
JsonStreamingExample.scala

This example uses the JsonSchemaEntities algebra to derive the JSON serialization format from a JSON schema, which can also be reused by the OpenAPI interpreter.

Eventually, mix a JsonSchemaEntities interpreter of your choice to turn the JSON schemas into proper JSON codecs. For instance, for Akka-Http and using Circe:

import akka.stream.scaladsl.Source
import endpoints.akkahttp.server
import endpoints.akkahttp.server.Streaming.Settings

class JsonStreamingExampleServer(val settings: Settings)
  extends JsonStreamingExample
    with server.JsonStreaming
    with server.JsonSchemaEntities {

  val routes =
    ticks.implementedBy(_ => Source.tick(0.seconds, 1.second, ()))

}
JsonStreamingExampleServer.scala

Custom Serialization Format

Support for other serialization formats can be added by defining an operation returning a Chunks[A] value. For instance, the Protocol Buffers format would be supported as follows (using protoless):

import endpoints.algebra
import io.protoless.messages.{Decoder, Encoder}

trait ProtobufStreaming extends algebra.Streaming {

  /** Chunks containing values of type `A`, serialized with the given protobuf codec */
  def protobufChunks[A : Encoder : Decoder]: Chunks[A] { type Serialized = Array[Byte] }

}

And then, the protobufChunks operation would have to be implemented on each interpreter.
